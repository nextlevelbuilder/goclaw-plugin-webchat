Proxy + API Key Auth

When PROXY_API_KEY is set on the proxy server, only backend services that provide a matching key can connect. This is for server-to-server authentication, not browser clients.

When to Use

Express/Node.js backend Multi-tenant Backend-to-proxy auth
⚠ Security: The PROXY_API_KEY must never be exposed in client-side JavaScript. It is used only by backend services (Express, nginx, etc.) that connect to the proxy on behalf of browser clients. Browser clients connect through your backend, which adds the API key server-side.

Architecture

1 Browser connects to your backend (Express, nginx, etc.)
2 Your backend connects to proxy with X-API-Key header or ?apiKey=xxx
3 Proxy verifies API key, then relays to GoClaw Gateway with GOCLAW_TOKEN

Proxy Server Setup

# server/.env
GOCLAW_URL=wss://your-gateway.goclaw.sh/ws
GOCLAW_TOKEN=your-gateway-token
PROXY_API_KEY=my-secret-proxy-key

Backend Integration (Express example)

// Your Express backend — API key stays server-side
const PROXY_API_KEY = process.env.PROXY_API_KEY;

// Proxy WebSocket from browser → proxy server
wss.on('connection', (clientWs) => {
  const upstream = new WebSocket(
    `ws://proxy:3100/ws?apiKey=${PROXY_API_KEY}`
  );
  // relay frames between client and proxy...
});

Client Code (no API key!)

// Browser connects to YOUR backend, not directly to proxy
GoClaw.init({
  url: 'wss://your-backend.com/ws',
  title: 'Chat with us',
});
ⓘ This example connects directly to the proxy for demo purposes (no API key required). In production, set PROXY_API_KEY and route through your backend.
